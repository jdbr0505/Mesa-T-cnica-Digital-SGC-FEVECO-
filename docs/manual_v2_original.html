Manual de Uso e Ingeniería del Sistema

Guía Metodológica para la Automatización de Competencias de Toros Coleados

1. Arquitectura General del Sistema

El software ha sido diseñado bajo patrones arquitectónicos modernos de desarrollo industrial, garantizando desacoplamiento, alta cohesión y robustez matemática. Siguiendo los principios de ingeniería de Roger Pressman, la solución implementa una arquitectura limpia distribuida en capas:

  • Capa de Presentación (Frontend): Desarrollada con interfaces nativas semánticas y estilizadas interactivas para una respuesta inmediata.
  • Capa de Lógica de Negocios (Backend): Motorizado mediante un núcleo algorítmico robusto que gestiona las reglas oficiales de la Federación Venezolana de Coleo (FEVECO).
  • Capa de Persistencia (Base de Datos): Un modelo híbrido relacional SQL para auditorías permanentes de resultados históricos y un nodo NoSQL en tiempo real para la transmisión inmediata a pantallas de la manga.

2. Módulo 1: Gestión e Inscripción de Atletas

Este módulo digitaliza el proceso clásico de registro. Captura las entidades esenciales mapeadas en el modelo relacional:

  1. Coleador: Identificado por su Cédula de Identidad como clave primaria inequívoca.
  2. Equino (Caballo): Registrado individualmente por nombre y trazabilidad de rendimiento.
  3. Propietario / Entidad: Asignación lógica que permite auditorías de procedencia del animal.
Garantía de Integridad Referencial
La base de datos bloquea automáticamente inscripciones duplicadas de un mismo atleta o equino dentro del mismo torneo, eliminando los errores humanos comunes de las planillas en papel.

3. Módulo 2: Sorteo Estocástico Inicial

Para la primera salida, el sistema implementa un algoritmo de aleatoriedad controlada. Este proceso asegura transparencia total. Técnicamente, toma el arreglo indexado de coleadores inscritos y aplica un algoritmo de barajado uniforme.

// Pseudocódigo del Motor de Aleatoriedad (1ra Salida) Función GenerarSorteoInicial(ListaColeadores) { Para i desde ListaColeadores.Longitud - 1 hasta 1: j = GenerarEnteroAleatorio(0, i) Intercambiar(ListaColeadores[i], ListaColeadores[j]) Retornar AgruparEnBloquesDeCuatro(ListaColeadores) }

4. Módulo 3: Captura de Cómputos Métricos

Este módulo duplica digitalmente la estructura analógica de las planillas oficiales de manga de FEVECO. La mesa técnica introduce los valores recolectados por los jueces de manga durante los 4 minutos reglamentarios del turno:

  • Coleadas Efectivas (Puntaje Positivo): Suma directa de puntos al atleta.
  • Coleadas Nulas (Puntaje Negativo): Penalización algorítmica inmediata en la tabla general.
  • Amonestaciones y Saques de Puerta: Métricas de control de disciplina que actúan como variables de desempate secundario.

5. Módulo 4: Algoritmo de Sorteo Dinámico (Segunda Salida)

A diferencia del primer sorteo, la segunda salida es completamente determinista y dinámica. El sistema ejecuta una consulta estructurada que ordena a todos los participantes por su puntuación neta acumulada de forma descendente.

Posteriormente, el algoritmo redistribuye a los coleadores emparejando a los de alto rendimiento en los primeros turnos ("Turnos de Élite") y así sucesivamente, lo cual eleva la competitividad del espectáculo deportivo de acuerdo con los estándares internacionales de gestión de torneos.

6. Guía de Operación para la Exposición

Paso 1: Inicie la presentación mostrando el módulo de registro. Agregue un coleador ficticio para demostrar que la interfaz responde en tiempo real.

Paso 2: Diríjase a la pestaña de "Sorteo Primera Salida" y presione el botón de ejecución para demostrar cómo el sistema automatiza la creación de turnos de manera limpia y sin intermediarios.

Paso 3: Simule la carga de cómputos en la mesa técnica y guarde. Muestre de inmediato la tabla de posiciones consolidadas para evidenciar el cálculo automático de puntajes netos.

Paso 4: Culmine la demostración ejecutando el Sorteo Dinámico para mostrar cómo el software reacciona inteligentemente a los datos de la primera fase.
